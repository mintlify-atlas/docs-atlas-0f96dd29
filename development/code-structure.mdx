The extension follows a clean, modular structure with each component in its own directory. This page documents the purpose of each file and the key functions that power the extension.

## Content Script

### `content/content.js`

The heart of the extension. Runs on every GitHub page and handles button injection.

<CodeGroup>

```javascript parseGitHubRepo()
function parseGitHubRepo() {
  const path = window.location.pathname.replace(/^\//"", "").replace(/\/$/"", "");
  const segments = path.split("/");
  if (segments.length < 2) return null;

  const owner = segments[0];
  const repo = segments[1];

  const reserved = [
    "settings", "marketplace", "explore", "topics", "trending",
    "collections", "events", "sponsors", "login", "signup",
    "notifications", "new", "organizations", "features", "pricing",
  ];
  if (reserved.includes(owner)) return null;

  return { owner, repo, fullName: `${owner}/${repo}` };
}
```

```javascript getMintlifyUrl()
function getMintlifyUrl(owner, repo) {
  return `${MINTLIFY_BASE}/${owner}/${repo}`;
}
```

```javascript injectActionButton()
function injectActionButton(owner, repo) {
  if (document.getElementById(ACTION_BTN_ID)) return;

  const actionList = document.querySelector(
    'ul.pagehead-actions, .d-flex.gap-2 .d-flex, #repository-details-container ul'
  );

  if (!actionList) return;

  const li = document.createElement("li");
  li.style.display = "inline-block";
  li.id = ACTION_BTN_ID;

  const link = document.createElement("a");
  link.href = getMintlifyUrl(owner, repo);
  link.target = "_blank";
  link.rel = "noopener noreferrer";
  link.className = "mintlify-repo-action-btn";
  link.innerHTML = `
    <span>View Docs on Mintlify</span>
    <svg>...</svg>
  `;

  li.appendChild(link);
  actionList.insertBefore(li, actionList.firstChild);
}
```

</CodeGroup>

### Key Functions Explained

#### `parseGitHubRepo()` <small>content/content.js:7</small>

Extracts the repository owner and name from the current URL.

**Returns:**
- `{ owner, repo, fullName }` if valid repository page
- `null` if not a repository or reserved GitHub path

**Reserved Owner Names:**

To avoid false positives, the function filters out GitHub's special paths:

<Tabs>
  <Tab title="Account Pages">
    - `settings`
    - `login`
    - `signup`
    - `notifications`
  </Tab>
  <Tab title="Discovery">
    - `explore`
    - `topics`
    - `trending`
    - `collections`
  </Tab>
  <Tab title="Other">
    - `marketplace`
    - `events`
    - `sponsors`
    - `organizations`
    - `features`
    - `pricing`
  </Tab>
</Tabs>

This ensures the extension only activates on actual repository pages like `github.com/facebook/react`, not on `github.com/explore` or `github.com/settings`.

#### `getMintlifyUrl()` <small>content/content.js:25</small>

Constructs the Mintlify documentation URL.

**Pattern:** `https://www.mintlify.com/{owner}/{repo}`

#### `injectActionButton()` <small>content/content.js:29</small>

Creates and injects the "View Docs on Mintlify" button into GitHub's UI.

**Target selectors:**
- `ul.pagehead-actions` - Standard repo action list
- `.d-flex.gap-2 .d-flex` - Alternative layout
- `#repository-details-container ul` - Mobile/responsive layout

<Note>
  The function checks for existing button via `ACTION_BTN_ID` to prevent duplicate injections.
</Note>

#### `cleanup()` <small>content/content.js:60</small>

Removes the injected button from the DOM. Called when:
- Extension is disabled
- User navigates to a non-repository page
- Before re-injection during SPA navigation

#### `init()` <small>content/content.js:65</small>

Main initialization function that:
1. Checks if extension is enabled via `chrome.storage.sync`
2. Parses the current URL
3. Injects the button if valid

## Background

### `background/service-worker.js`

Minimal service worker with two responsibilities:

```javascript background/service-worker.js
chrome.runtime.onInstalled.addListener(() => {
  chrome.storage.sync.set({ enabled: true });
});

chrome.action.onClicked.addListener((tab) => {
  if (!tab.url || !tab.url.startsWith("https://github.com/")) return;

  const path = new URL(tab.url).pathname.replace(/^\//"", "").replace(/\/$/"", "");
  const segments = path.split("/");
  if (segments.length < 2) return;

  const owner = segments[0];
  const repo = segments[1];
  const mintlifyUrl = `https://www.mintlify.com/${owner}/${repo}`;

  chrome.tabs.create({ url: mintlifyUrl });
});
```

## CSS Styling Approach

The extension uses a **scoped styling** strategy:

- **Inline button** (`content.css`) uses specific class names (`mintlify-repo-action-btn`) to avoid conflicts with GitHub's styles
- **Popup** (`popup.css`) uses a clean slate with reset styles since it's an isolated context
- **`!important` flags** ensure color overrides work on GitHub's complex CSS

<Warning>
  Avoid generic class names in content scripts. GitHub's CSS could override your styles or vice versa.
</Warning>

## Constants and Configuration

```javascript content/content.js
const MINTLIFY_BASE = "https://www.mintlify.com";
const ACTION_BTN_ID = "mintlify-action-btn";
```

- **`MINTLIFY_BASE`**: Base URL for all Mintlify docs links
- **`ACTION_BTN_ID`**: Unique ID to prevent duplicate button injection

## IIFE Pattern

Both `content.js` and `popup.js` use Immediately Invoked Function Expressions:

```javascript
(function () {
  "use strict";
  // Extension code
})();
```

This creates a private scope and prevents variable pollution in the global namespace.
