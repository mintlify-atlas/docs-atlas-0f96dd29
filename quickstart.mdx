This guide walks you through using the extension from installation to viewing your first auto-generated documentation.

<Note>
  This quickstart assumes you've already [installed the extension](/installation). If not, complete the installation first.
</Note>

## View documentation for a repository

<Steps>
  <Step title="Navigate to a GitHub repository">
    Open any public GitHub repository in your browser. For this example, we'll use React:
    
    ```text
    https://github.com/facebook/react
    ```
    
    The extension works with any public repository, including:
    - Popular open-source projects
    - Your own public repositories
    - Organization repositories
    - Forked repositories
  </Step>
  
  <Step title="Locate the action button">
    Once the repository page loads, look for the **View Docs on Mintlify** button in the repository's action bar. It appears alongside the Watch, Fork, and Star buttons near the top of the page.
    
    The button includes:
    - Text: "View Docs on Mintlify"
    - An external link icon
    - Consistent styling with GitHub's native buttons
    
    <Tip>
      If you don't see the button immediately, wait a moment for the page to fully load, then refresh if needed.
    </Tip>
  </Step>
  
  <Step title="Click to view documentation">
    Click the **View Docs on Mintlify** button. This opens a new browser tab with the auto-generated documentation for the repository.
    
    The URL transforms from:
    ```text
    github.com/facebook/react
    ```
    
    To:
    ```text
    mintlify.com/facebook/react
    ```
  </Step>
  
  <Step title="Explore the documentation">
    On the Mintlify documentation page, you can:
    - Browse the codebase structure
    - Read function and class documentation
    - Search for specific APIs or features
    - Navigate through different files and modules
  </Step>
</Steps>

## Using the extension popup

The extension also provides a popup interface for quick access and settings:

<Steps>
  <Step title="Open the popup">
    Click the Mintlify extension icon in your browser toolbar (usually in the top-right corner).
  </Step>
  
  <Step title="View current repository">
    If you're on a GitHub repository page, the popup displays:
    - The repository name (e.g., `facebook/react`)
    - A clickable button to open documentation
    
    If you're not on a repository page, it shows: "Open a GitHub repo first"
  </Step>
  
  <Step title="Open documentation from popup">
    Click the main button in the popup to open the Mintlify documentation in a new tab, just like clicking the injected button on the GitHub page.
  </Step>
  
  <Step title="Toggle the extension">
    Use the "Extension enabled" switch at the bottom of the popup to:
    - **Disable**: Removes the action button from all GitHub pages
    - **Enable**: Shows the action button on GitHub repository pages
    
    Your preference is saved automatically.
  </Step>
</Steps>

## Code example: URL parsing

Under the hood, the extension parses GitHub URLs to extract the repository information. Here's how it works:

```javascript
function parseGitHubRepo() {
  const path = window.location.pathname.replace(/^\//,"").replace(/\/$/,"");
  const segments = path.split("/");
  if (segments.length < 2) return null;

  const owner = segments[0];
  const repo = segments[1];

  const reserved = [
    "settings", "marketplace", "explore", "topics", "trending",
    "collections", "events", "sponsors", "login", "signup",
    "notifications", "new", "organizations", "features", "pricing",
  ];
  if (reserved.includes(owner)) return null;

  return { owner, repo, fullName: `${owner}/${repo}` };
}
```

This function:
1. Extracts the pathname from the current URL
2. Splits it into segments (owner and repository name)
3. Validates that it's a real repository (not a GitHub system page)
4. Returns the parsed repository information

<Note>
  The extension automatically excludes GitHub system pages like Settings, Marketplace, and Explore to avoid showing the button on non-repository pages.
</Note>

## Keyboard workflow tip

For power users, you can streamline your workflow:

1. Use `Ctrl+L` (Windows/Linux) or `Cmd+L` (Mac) to focus the address bar
2. Type your GitHub repository URL
3. Press `Enter` to navigate
4. Click the **View Docs on Mintlify** button
5. Switch between GitHub and Mintlify tabs with `Ctrl+Tab`

<Tip>
  Pin the Mintlify extension to your toolbar for quick access to the popup by right-clicking the extension icon and selecting "Pin".
</Tip>
